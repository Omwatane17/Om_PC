── FAIRNESS ENGINE ─────────────────────────────────────────────────────────────

[✅] Sensitive Attribute Auto-Detector
      - Stage 1: Regex keyword matching (gender, race, age, zip, etc.)
      - Stage 2: Low-cardinality value distribution analysis
      - Catches: gender, race, age, education, ZIP, nationality, etc.

[✅] 6 Fairness Metrics (computed per protected attribute):
      1. Demographic Parity Difference
      2. Disparate Impact Ratio  (EEOC 4/5ths Rule)
      3. Equalized Odds Difference
      4. Equal Opportunity Difference
      5. Predictive Parity Difference
      6. Treatment Equality (FN/FP ratio)

[✅] Regulation-aware thresholds:
      - Generic / EU AI Act / EEOC / CFPB

[✅] Risk Scoring Formula
      - Weighted composite → 0–100 score
      - Risk levels: LOW / MEDIUM / HIGH / CRITICAL

[✅] SHAP Feature Importance
      - LogisticRegression + LinearExplainer
      - Per-group SHAP breakdown
      - Correlation-based fallback if SHAP fails

[✅] Counterfactual Generator
      - Flips protected attribute values for denied individuals
      - Shows score delta and decision reversal
      - Sorted by impact magnitude


── LLM INTEGRATION ─────────────────────────────────────────────────────────────

[✅] Claude API integration (claude-opus-4-5)

PRIORITY 2 — ADDITIONAL FAIRNESS METRICS ──────────────────────────────────────

[🔲] Calibration Difference
      → Requires probability scores (not just binary labels)
      → Formula: |P(Y=1|ŷ=s, A=a) − P(Y=1|ŷ=s, A=b)|

[🔲] Individual Fairness Score
      → Lipschitz condition check over sampled pairs
      → Computationally expensive; sample 500 pairs

[🔲] Counterfactual Fairness Score
      → P(ŷ_CF=1) vs P(ŷ=1) after attribute flip aggregated

[🔲] Intersectional Analysis
      → Cross-group disparity (e.g., Black+Female vs White+Male)
      → Requires combinatorial grouping logic
